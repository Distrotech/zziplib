You need... content pages.

The content pages should not contain anything usually needed for the html envelope of a webpages - do not use "<html>", "<head>" "<body>". For a fact, you can even omit the "<title>" tag as it is purely optional for mksite.sh - the shell script will scan the content page looking for the html header markups "<h1>", "<h2>",... "<h6>" and use the first one for the page title.

Example:
<H4> Lorem Ipsum </H4>
<P>

That's all - just a header line and some text.
Mark them as "H4"eader and "P"aragraphs.
The rest is done in the sitefile.

The text above is used quite usually in typography to denote a blind text, i.e. text that is not readily available. A typograph may create a style for a book and in order to show it around some helper text must be inserted as a placeholder for the real text to be added later. To use "lorem ipsum" is a very old habit going back right to the 1500s as the first days of "industrial" book printing. The text itself is latin, an excerpt of "De Finibus et Malorum" by the great Cicero - it speaks about ethics and the first line translates roughly to "There is no one who loves pain itself, who seeks after it and wants to have it, simply because it is pain.". Actually, you may use any other blind text at hand but take notice that modern Desktop Publishing systems will detect the words "Lorem Ipsum" and before sending it to the printer (creating ten thousand copies) it will warn harshly "hey there is some blind text left". And there: use Lorem Ipsum.
